1. Executive Summary

Project: RCPL operates two independent QR Code systems within a single codebase, each serving a different regulatory requirement for their rPET product line. The system is fully independent from existing RLL/RCPM QR infrastructure.
System 1

FSSAI Product Information
QR code maps to /. Consumers enter a 1–2 character batch code from the bottle neck to retrieve manufacturer name, vendor address, and FSSAI compliance license number.
System 2 NEW v1.1

rPET Plastic Recycle Declaration
QR code maps to /rpet. Consumers enter the full bottle neck code (e.g. MBB306C) to verify the recycled plastic content percentage, manufacturing location, and PWM registration number — as mandated by Plastic Waste Management Rules.

2. Problem Statement

Consumer Problem — FSSAI
Consumers have no easy way to verify rPET product authenticity, manufacturer details, or FSSAI compliance. Regulatory requirements mandate this information be accessible via QR code.
Consumer Problem — PWM Rules
India's Plastic Waste Management Rules require manufacturers to declare the recycled plastic content (rPET %) on packaging and make it verifiable. A QR-accessible declaration fulfils this mandate.
Regulatory Team Problem: The existing vendor management system (RLL/RCPM) cannot be modified. A new independent system is needed that the regulatory team can update without IT dependency for each change.
Solution: Dual-system web application within a single monorepo. System 1 handles FSSAI batch code lookups. System 2 handles PWM recycle content declarations. Both share a single admin portal with separate sections. Each maps to its own QR Planet QR code URL.

Bottle Code Structure (System 2):
The combined_code is a concatenation of four lookup segments:

[ VendorCode ][ ProductVariantCode ][ SKUCode ][ RecycleCode ]

Example: MBB306C = Vendor M + Variant BB + SKU code 306 + Recycle code C
Each segment maps to a lookup table administered by the regulatory team.

DB File Schema (rpet.json)

{
  "products": [ ...RpetProduct[] ],
  "nextId": 4,
  "declarations": [ ...RpetDeclaration[] ],
  "declNextId": 3
}

12. Out of Scope

  • Integration with existing RLL/RCPM vendor management system
  • Consumer authentication or account creation
  • QR code generation (handled by QR Planet)
  • Email/SMS notifications
  • Multi-language support (Hindi, regional languages)
  • Native mobile app
  • Price or inventory information
  • Dynamic recycle code lookup table management via UI (P2 item — currently hardcoded seed data)

14. Acceptance Criteria

System 1 — FSSAI

  • Consumer can scan QR, enter "AA", see full FSSAI product details within 10 seconds on mobile
  • Inactive product code returns "not available" (not a raw error)
  • Non-existent code returns friendly "product not found" with back button
  • Admin can create a new product record with all required fields
  • Admin can edit vendor address and FSSAI license number
  • Admin can deactivate a product (soft delete with confirmation modal)
  • FSSAI validation rejects anything that is not exactly 14 digits

System 2 — rPET Declaration NEW v1.1

  • Consumer can scan QR, enter "MBB306C", see recycle declaration with % and PWM details within 10 seconds
  • Recycle content progress bar renders correctly for values between 0% and 100%
  • Inactive declaration code returns friendly error with "Try Again" button
  • Admin can create a new declaration with combined_code, product name, manufacturing location, recycle %, and PWM reg no
  • Admin can edit recycle content and PWM registration number
  • Admin can deactivate a declaration (soft delete with confirmation modal)
  • Duplicate combined_code on creation returns 409 conflict error in admin form
  • Admin sidebar shows both "FSSAI Products" and "rPET Declarations" sections
  • Admin footer provides quick links to both consumer-facing pages

Shared

  • All pages render correctly on iPhone SE (375px) screen width
  • CI/CD pipeline passes: lint → test → build → security audit
  • Both landing page URLs are provided for QR Planet mapping
  • GitHub repository is up to date with all changes pushed to main
