1. Executive Summary

Project: RCPL requires a QR Code-based product information system for their rPET product line. A QR code is already purchased from QR Planet. This system provides three independent components: consumer landing page, product information display, and regulatory admin portal. The system must be fully independent from existing RLL/RCPM QR systems.

The key deliverable is a URL that RCPL's regulatory team maps to the QR Planet-hosted QR code. Consumers scan the QR code on the bottle, enter the batch code (first 1–2 characters printed on the bottle neck), and instantly see manufacturer and FSSAI compliance details on their mobile device.

2. Problem Statement

Consumer Problem
Consumers have no easy way to verify rPET product authenticity, manufacturer details, or FSSAI compliance. Regulatory requirements mandate this information be accessible via QR code.
Regulatory Team Problem
The existing vendor management system (RLL/RCPM) cannot be modified. A new independent system is needed that the regulatory team can update without IT dependency for each change.
Solution: Three-component web system (Consumer Landing Page → Batch Code Lookup → Admin Data Portal) deployed as an independent URL, mapped to the existing QR Planet QR code.

14. Acceptance Criteria

  • Consumer can scan QR, enter "AA", see full product details within 10 seconds on mobile
  • Inactive product code returns "information not available" (not a 404 error)
  • Non-existent code returns friendly "product not found" with back button
  • Admin can create a new product record with all required fields
  • Admin can edit vendor address and FSSAI license number
  • Admin can deactivate a product (soft delete with confirmation)
  • FSSAI validation rejects anything that is not exactly 14 digits
  • All pages render correctly on iPhone SE (375px) screen width
  • CI/CD pipeline passes: lint → test → build → security audit
  • Landing page URL is provided for QR Planet mapping
